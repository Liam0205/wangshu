## 当前缺口

- **arena ABI 字段级 spec 缺失** — `docs/design/roadmap.md` (§8) 只给概念:类型化扁平列 + 字符串区 + presence bitmap。**字符串区编码、presence bitmap 位布局、`Program.Call` 的 `args` 与 arena 的精确关系**均未定义。定稿后应在 [[embedding-contract]] 补字段级 spec。

- **P2 无独立量化验收门槛** — 源文档对 P2(分层桥)只给人力估算与技术内容,未给倍率/验收(定位为基建)。已在 [[evolution-roadmap]] 如实标注「文档未给独立量化门槛」,非笔误。

- **倍率两套坐标系** — 流水线图倍率(P3「4-8x」、P4「trace 收益 ~70%」)与阶段正文验收门槛(P3「相对 P1 再 ≥2x」、P4「≥ LuaJ-luajc 档」)不在同一坐标系。已在 [[evolution-roadmap]] 分列两栏并加坐标系警告;若后续源文档统一口径,应回收此提示。

- **人力估算口径不统一** — P1/P2/P3 用「人月」,P4/P5 用「人年」且 P5 标「开放式」;无团队规模或日历时间换算。如实保留。

- **校准测量原始数据未入库** — §附注称原始数据与方法留存于发起方仓库工作区,正式立项后整理为 `benchmarks/baseline/`;当前本仓库无此目录。入库后可在 [[design-premises]] / [[glossary]] 增加可追溯链接。

- **无任何代码可交叉验证** — 全部内容均为设计意图。一旦开始实现,需新建 `guides/`(构建/测试/差分 fuzz 工作流)并把架构文档从「规划」措辞更新为「实现现状」。
